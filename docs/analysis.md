### 1.2 十五健康维度概览 (15 Dimensions)

### 1.3 问题结构 (30+2 Questions)

- **主要题目**：30 道，每个维度严格分配 2 道（共 15 维度 x 2 = 30）
- **特殊题目**：2 道吸烟门控题（smoke_gate_q1 + smoke_gate_q2）
- **选项设计**：主要题目每道 3 个选项，对应数值 1/2/3（1=健康行为，3=高风险行为）；smoke_gate_q1 有 4 个选项
- **展示方式**：30 道主题随机打乱，smoke_gate_q1 随机插入其中

## 3. 风险画像空间分析

### 3.1 分析方法

将每个画像的 15 维模式串转化为数值向量（L=1, M=2, H=3），计算所有 C(25,2)=300 个画像对之间的曼哈顿距离：

$$d(A,B) = \sum_{i=1}^{15} |A_i - B_i|$$

理论距离范围：0（完全相同）至 30（每维度都相差 2）。

### 3.2 距离统计摘要

| 统计量 | 值 |
|--------|-----|
| 总配对数 | 300 |
| 最小距离 | **1** |
| 最大距离 | **24** |
| 理论最大 | 30 |

> 实际最大距离 24 仅达到理论上限的 80%，说明画像空间并未覆盖到极端对立的类型。

- **H3（三高预备军）**：风险分和 41——需要用户在绝大多数维度都得高分（高风险行为）
- **DEAD（全面崩塌者）**：风险分和 20——需要用户在绝大多数维度都得低分（健康行为）

### 3.6 画像聚类结构

**聚类 A - "高压奋斗群"**（互距 2~6）：
- H3（三高预备军）、STRESS（高压打工魂）、GRIND（奋斗鸡血人）、SED（久坐办公族）
- 共同特征：D1=H, D2=H, M1=H, R2=H（饮食不规律、高压、不体检）

**聚类 B - "心理困扰群"**（互距 2~5）：
- ANX（焦虑星人）、OVERTHINK（过度思考者）、CYNIC（愤世健康盲）
- 共同特征：M1=H, M2=H/M, M3=H（高压、情绪调节困难、高焦虑）

**聚类 C - "低关注脆弱群"**（互距 1~6）：
- JOKER（佛系躺平人）、WRECK（身体报废者）、CLUELESS（健康小白）、DEAD（全面崩塌者）
- 共同特征：多数维度为 L，R3 偏向 L（健康关注度低）

**聚类 D - "中间温和群"**（互距 5~9）：
- MEH（无所谓人）、CAREGIVER（过度操劳者）、SLOTH（懒懒兽）
- 共同特征：多数维度为 M，没有极端取值

**孤立画像**：
- CAREFREE（心大快乐人）：独特的 E1=H + E3=L + R1=H 组合（爱运动但饮酒）
- REBEL（叛逆放纵者）：独特的 R1=H + 几乎全 L 的其他维度（只有烟酒高风险）
- LONER（孤僻者）：独特的 S2=H + E3=H 但其余全低（作息乱且久坐）

## 4. 特殊机制

### 4.1 吸烟门控流程 (Smoke Gate)

吸烟门控是一个两阶段条件触发机制，用于解锁隐藏画像 SMOKER（老烟枪）：
smoke_gate_q1
"您平时有什么生活习惯？"
/ | |
健康饮食 规律运动 吸烟 饮酒
(v=1) (v=2) (v=3) (v=4)
| | | |
v v | v
[忽略] [忽略] | [忽略]
v
smoke_gate_q2
"您的吸烟习惯是？"
/
偶尔社交场合 每天一包以上
(v=1) (v=2)
| |
v v
[忽略] SMOKER 激活！
|
v
覆盖正常匹配结果
正常最佳匹配降为副结果
```

**设计要点**：
1. smoke_gate_q1 随机插入 30 题中，用户无法预判
2. 仅选择"吸烟"（v=3）才会触发 q2，其他选项不产生后续
3. q2 的选项设计极端化："偶尔社交场合"不触发，"每天一包以上"才触发
4. 一旦 SMOKER 激活，完全覆盖正常匹配流程，正常最佳匹配仅作为次要展示
5. smoke_gate_q1 和 q2 都**不参与**维度评分，仅用于门控判定

### 4.2 BALANCED 均衡健康人兜底机制

当用户的 15 维向量与所有 25 个标准画像的**最佳匹配相似度低于 60%** 时，系统强制分配 BALANCED（均衡健康人）：
相似度计算公式：
similarity = max(0, round((1 - distance/30) * 100))

触发条件：
similarity < 60%
=> (1 - distance/30) * 100 < 60
=> distance/30 > 0.4
=> distance > 12

即：当用户向量到最近画像的曼哈顿距离 > 12 时，触发兜底。
```

**触发概率分析**：

距离 > 12 意味着用户的 15 维向量与所有 25 个画像都相差甚远。考虑到：
- 25 个画像在 15 维空间中已有一定覆盖度
- 最中心的画像（如 MEH、CAREGIVER）到空间中心的距离较小
- 用户答题产生的向量有 3^15 = 14,348,907 种理论组合

实际上，由于画像设计覆盖了高/中/低各区域，触发 BALANCED 的概率较低。最可能触发的用户画像是：答题模式极端不一致（某些维度极健康、某些极高风险，且组合方式与任何已定义画像都不匹配）。

### 4.3 反向计分题目

设计时需注意：某些题目需要反向计分，以确保"健康行为"对应低分值（1）。例如：

| 题号 | 维度 | 选项值顺序 | 说明 |
|------|------|------------|------|
| q6 | D3 | 3,2,1 | "我很少吃蔬菜"——认同=3（高风险），不认同=1（健康） |
| q14 | S2 | 3,2,1 | "我经常熬夜"——认同=3（高风险），不认同=1（健康） |
| q16 | E2 | 1,2,3 | "我会定时起身活动"——认同=1（健康），不认同=3（高风险） |

**设计意图**：
1. 防止用户全选同一位置
2. 语义校准：某些题目的文字方向与健康方向相反
3. 降低社会期望偏差

## 5. 评分算法详解

### 5.1 完整步骤拆解

#### 步骤一：维度求和

用户完成 30 道主要题目后，按维度将 2 道题的选项值相加：
D1 = q1.value + q2.value (范围 2~6)
D2 = q3.value + q4.value
D3 = q5.value + q6.value
E1 = q7.value + q8.value
E2 = q9.value + q10.value
E3 = q11.value + q12.value
S1 = q13.value + q14.value
S2 = q15.value + q16.value
S3 = q17.value + q18.value
M1 = q19.value + q20.value
M2 = q21.value + q22.value
M3 = q23.value + q24.value
R1 = q25.value + q26.value
R2 = q27.value + q28.value
R3 = q29.value + q30.value
```

#### 步骤二：分级映射

将每个维度的原始分（2~6）映射为 L/M/H 三级：

| 原始分 | 等级 | 数值 | 风险含义 | 可能的答题组合 |
|--------|------|------|----------|----------------|
| 2 | L | 1 | 健康 | 1+1 |
| 3 | L | 1 | 健康 | 1+2, 2+1 |
| 4 | M | 2 | 中风险 | 1+3, 2+2, 3+1 |
| 5 | H | 3 | 高风险 | 2+3, 3+2 |
| 6 | H | 3 | 高风险 | 3+3 |

> **注意**：L 覆盖 2 种原始分（2,3），M 仅覆盖 1 种（4），H 覆盖 2 种（5,6）。这使得 M 的区间最窄——用户必须恰好得 4 分才能落入 M，概率上 M 比 L 和 H 更难获得。

#### 步骤三：构建用户风险向量

将 15 个维度的等级数值化组成向量：
user_vector = [D1_num, D2_num, D3_num, E1_num, E2_num, E3_num,
S1_num, S2_num, S3_num, M1_num, M2_num, M3_num,
R1_num, R2_num, R3_num]
```

#### 步骤四：计算曼哈顿距离

对每个标准画像，解析其模式串并计算距离：
以 H3（三高预备军）为例
```
type_pattern = "HHH-HHH-MHH-HMM-HHL"
type_vector = [3,3,3,3,3,3,2,3,3,3,2,2,3,3,2] # 去掉横杠，H=3,M=2,L=1

distance = sum(|user_vector[i] - type_vector[i]|) for i = 0..14
exact = count(user_vector[i] == type_vector[i]) for i = 0..14
similarity = max(0, round((1 - distance/30) * 100))
```

#### 步骤五：排序与选择
结果列表按以下优先级排序：

distance ASC（距离越小越好）

exact DESC（精确命中维度越多越好）

similarity DESC（相似度越高越好）

取排序后的第一个作为最佳匹配。
```

#### 步骤六：特殊规则覆写
if smoke_gate_q2 == 2:
result = SMOKER
secondary = 正常最佳匹配
elif best_similarity < 60%:
result = BALANCED
else:
result = 正常最佳匹配
```

新画像需要一个 15 维的 L/M/H 模式串，格式为 `XXX-XXX-XXX-XXX-XXX`（每段 3 个字符，对应一个模型的 3 个维度）。

- 模式串的风险分和（L=1,M=2,H=3 之和）应考虑空间覆盖：当前 20~41 范围内，可在空缺区域填充

6.3 调整评分参数
修改分级阈值：

当前阈值（2 题/维度，原始分 2~6）：

```
L（健康）: score <= 3
M（中风险）: score == 4
H（高风险）: score >= 5
```

可调整为更宽的 M 区间以增加 M 出现概率：

```javascript
// 方案 A：扩大 M 区间
L: score <= 2
M: score == 3 or score == 4
H: score >= 5

// 方案 B：三等分（当前方案）
L: score <= 3
M: score == 4
H: score >= 5
```
注意：调整阈值会改变用户向量的分布，可能需要同步调整画像模式串或增加新画像来覆盖新的分布。

修改相似度计算：

当前公式 similarity = max(0, round((1 - distance/30) * 100))

分母 30 是理论最大距离（15 维 x 每维最大差 2）

可以引入加权距离：weighted_distance = sum(w[i] * |user[i] - type[i]|)，让某些维度的重要性更高（如让 R2 体检频率权重降低，因为区分力弱）

修改 BALANCED 兜底阈值：

当前阈值为 similarity < 60%（即 distance > 12）。

提高阈值（如 70%）：更多用户会触发兜底，使兜底更常见

降低阈值（如 50%）：减少兜底触发，但可能导致低匹配度的不佳体验
